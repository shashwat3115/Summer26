# Q13. Arduino Pin Types

The **Arduino UNO** provides different types of pins for interfacing with sensors, actuators, and communication modules. Each pin type serves a specific purpose in embedded systems and IoT applications.

| Pin Type | Description | Example | Real IoT Use Case |
|-----------|-------------|---------|-------------------|
| **Digital Input** | Reads binary signals (`HIGH` or `LOW`) from external devices such as switches or sensors. | Push button connected to **D2** | Detecting door open/close status in a smart home security system. |
| **Digital Output** | Sends binary signals (`HIGH` or `LOW`) to control external devices. | LED connected to **D13** | Turning a relay ON/OFF to control home appliances remotely. |
| **Analog Input** | Reads analog voltages (0–5 V) using the built-in **10-bit ADC**, producing values from **0–1023**. | TMP36 temperature sensor connected to **A0** | Monitoring room temperature for an IoT weather station. |
| **PWM Output** | Generates Pulse Width Modulated (PWM) signals using `analogWrite()`. Available on pins **3, 5, 6, 9, 10, and 11**. | LED brightness control on **D9** | Controlling the speed of a DC fan based on temperature readings. |
| **I2C / SPI Pins** | Used for communication with peripherals. **I2C:** SDA (A4), SCL (A5). **SPI:** SS (D10), MOSI (D11), MISO (D12), SCK (D13). | OLED display (I2C) or SD card module (SPI) | Collecting sensor data and displaying or storing it for IoT applications. |
