# Q9. Difference Between `git pull`, `git fetch`, and `git clone`

| Command | Description | When to Use |
|---------|-------------|-------------|
| **`git clone`** | Creates a complete local copy of a remote repository, including all files and commit history. | Use when downloading a repository for the first time. |
| **`git fetch`** | Downloads the latest commits and changes from the remote repository without modifying the local working branch. | Use when you want to review remote changes before merging them. |
| **`git pull`** | Downloads changes from the remote repository and automatically merges them into the current local branch. (`git pull = git fetch + git merge`) | Use when you want to update your local repository with the latest changes. |

## Summary

- **`git clone`** → Creates a new local copy of a remote repository.
- **`git fetch`** → Retrieves remote updates without changing local files.
- **`git pull`** → Retrieves and merges remote updates into the current branch.

# Q10. `.gitignore` File

## What is a `.gitignore` File?

A **`.gitignore`** file specifies which files and directories Git should ignore. Ignored files are not tracked or committed to the repository, helping keep the repository clean by excluding compiled files, temporary files, operating system files, and IDE configuration folders.
